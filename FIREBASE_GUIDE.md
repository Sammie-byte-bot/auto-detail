## How Firestore Works

Firestore organises data in a **hierarchy**:

```
Firestore Database
└── Collection  (like a table)
    └── Document  (like a row — has a unique ID)
        └── Fields  (key-value pairs — like columns)
            └── Sub-collection (optional nested collection)
```

**Key concepts:**
- A **Collection** is a group of Documents (e.g. `users`, `products`).
- A **Document** is a single record with named fields (e.g. a user profile).
- Documents can contain nested objects and arrays.
- Each Document has a unique **ID** (auto-generated or custom).

## How to Set Up Firebase

1. Go to [console.firebase.google.com](https://console.firebase.google.com) and create a project.
2. Click **Add App → Web** and copy your config.
3. Paste config into `js/firebase-config.js`.
4. Enable **Authentication** (Email/Password + Google) in the Firebase Console.
5. Create your **Firestore Database** (start in test mode, then apply `firestore.rules`).
6. Enable **Storage** if uploading images.

## Deploy to Firebase Hosting

```bash
npm install -g firebase-tools
firebase login
firebase init   # select Hosting, Firestore, Storage
firebase deploy
```

Your site will be live at `https://auto-detail-car-wash.web.app`
